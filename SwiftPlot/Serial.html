The Serial Interface is an interface for communicating with an Serial interface. Intended for use with Arduino, but useful for anything that sends data over a Serial Bus. The menubar is used for device selection, the detail view serves the interaction with a device.

Devices

Console

Protocol: JSON

To import data into SwiftPlot the data send over the serial bus must be in accordance with the protocol. The protocol exists over two parts:
- Console data
- Data Entries
Console data is any ASCII encoded string.
Data Entries are more complicated, they must be according to the correct format. This must still be send as an ASCII encoded string, but to be accepted as a data entry it must be of the following format: JSON
"[{\"key\":\"T\",\"value\":19.379999999999999},{"key":"t","value":1}]"
{} encapsulate a "key" : "value" pair in JSON format. Everything sent with a single "Entry: {"key"="string","value"="10"}" signal, will remain in that form, once recieved by SwiftPlot. * This means that every time "" is sent, an dictionary will be added to the array of DataEntries.
